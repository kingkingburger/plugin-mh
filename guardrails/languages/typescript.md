# TypeScript Guardrails

## 기본값

- 패키지 매니저와 임시 실행은 `bun` / `bunx`를 우선한다.
- 타입 검증은 호환되는 경우 `bunx tsgo --noEmit`을 우선한다.
- 테스트는 프로젝트 기본값을 따르되, 새 프로젝트는 Vitest를 기본 후보로 둔다.
- `strict` 타입을 전제로 설계한다.

## 코드 기준

- `any`는 외부 경계나 마이그레이션 중인 코드에만 제한적으로 허용한다.
- API 응답 타입, 컴포넌트 props, 저장 모델 타입을 서로 섞지 않는다.
- 파싱과 검증은 경계에서 한다. 내부 로직은 검증된 타입을 받는다.
- 비동기 흐름은 명시적으로 실패 경로를 처리한다.
- 공유 유틸은 실제 두 번째 호출 지점이 생기기 전까지 만들지 않는다.

## 테스트 기준

- 순수 로직은 단위 테스트를 먼저 둔다.
- API 계약 변경은 요청/응답 타입과 mock을 함께 갱신한다.
- UI 상태 변경은 사용자 관찰 동작을 기준으로 테스트한다.
- `<Teleport>`, modal, overlay, z-index, layout 변경은 브라우저 런타임 검증을 추가한다.

## 선호 명령

```bash
bun test
bun run test
bunx tsgo --noEmit
bun run build
```

프로젝트가 다른 명령을 이미 표준으로 쓰면 프로젝트 명령을 우선한다.
