# Evaluator Rubric — 깐깐한 평가 기준

> "문제를 찾아놓고도 '별로 심각하진 않네요. 통과' 이런 평가는 금지."
> 실제 사용자라면 불편할 모든 것을 지적하라.

## 평가 기준 (웹 앱)

### 1. 기능성 (30%)

| 점수 | 기준 |
|------|------|
| 10 | 모든 Acceptance Criteria가 완벽히 작동 |
| 7-9 | 핵심 기능 작동, 사소한 이슈 1-2개 |
| 4-6 | 일부 기능 미작동 또는 주요 버그 존재 |
| 1-3 | 핵심 기능 고장 (클릭 안 됨, 페이지 안 뜸 등) |
| 0 | 앱이 로드되지 않음 |

**Playwright 검증:**
1. `browser_navigate` → 앱 로드
2. `browser_snapshot` → DOM 구조
3. 각 AC별: `browser_click`, `browser_fill_form`, `browser_evaluate`
4. `browser_console_messages` → 에러 로그

### 2. 디자인 품질 (25%)

| 점수 | 기준 |
|------|------|
| 10 | 전문적이고 일관된 비주얼 시스템 |
| 7-9 | 깔끔하지만 약간의 불일치 |
| 4-6 | AI 슬롭 징후 일부 |
| 1-3 | 전형적 AI 슬롭 (보라색 그라데이션, 흰 카드, shadcn 기본값) |
| 0 | 스타일링 없음 또는 깨진 레이아웃 |

**Playwright 검증:**
1. `browser_take_screenshot` → 시각적 확인
2. `browser_resize` → 반응형 (1920px, 768px, 375px)
3. `browser_evaluate` → CSS 커스텀 속성/폰트

### 3. 독창성 (25%)

| 점수 | 기준 |
|------|------|
| 10 | 예상치 못한 창의적 접근 |
| 7-9 | 고유한 디자인 언어 또는 독특한 인터랙션 |
| 4-6 | 깔끔하지만 예상 가능한 구현 |
| 1-3 | 템플릿 수준 |
| 0 | 기본 HTML 또는 프레임워크 기본 테마 |

### 4. 완성도 (20%)

| 점수 | 기준 |
|------|------|
| 10 | 로딩 상태, 에러 처리, 엣지 케이스 모두 처리 |
| 7-9 | 주요 경로 완성, 사소한 엣지 케이스 미처리 |
| 4-6 | 핵심 경로만 작동 |
| 1-3 | 버튼만 있고 실제 기능 없음 |
| 0 | 미완성 |

**Playwright 검증:**
1. 빈 입력 제출 → 에러 메시지
2. 빠른 연속 클릭 → 중복 처리
3. `browser_network_requests` → API 에러

## 판정

| 총점 | 판정 | 액션 |
|------|------|------|
| 70-100 | **PASS** | 다음 스프린트 |
| 50-69 | **CONDITIONAL** | 주요 이슈만 수정 후 재평가 |
| 0-49 | **FAIL** | 전체 피드백 + 수정 + 재평가 |

## 가중 총점 계산

```
총점 = (기능성 × 3.0) + (디자인 × 2.5) + (독창성 × 2.5) + (완성도 × 2.0)
```

## CLI/API 평가 기준

| 기준 | 가중치 | 검증 |
|------|--------|------|
| 기능성 | 40% | Bash 명령 실행 + 출력 확인 |
| 에러 처리 | 25% | 잘못된 입력, 없는 파일 등 |
| 사용성 | 20% | 도움말, 직관적 인터페이스 |
| 완성도 | 15% | 엣지 케이스, 성능 |
