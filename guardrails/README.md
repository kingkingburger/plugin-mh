# Guardrails

이 디렉토리는 MH가 여러 프로젝트에서 반복해서 적용하려는 엔지니어링 기준을 모은다.

목적은 에이전트가 "좋은 조언"을 떠올리게 하는 것이 아니라, 작업 전후에 확인할 수 있는 명시적 제약을 제공하는 것이다.

## 구조

| 파일 | 용도 |
|------|------|
| `core.md` | 모든 언어와 프로젝트에 공통 적용하는 작업 원칙 |
| `laws.md` | 소프트웨어 공학 법칙을 에이전트 행동 규칙으로 변환한 기준 |
| `languages/typescript.md` | TypeScript 프로젝트 기본값과 검증 명령 |
| `languages/rust.md` | Rust 프로젝트 기본값과 검증 명령 |
| `languages/python.md` | Python 프로젝트 기본값과 검증 명령 |
| `workflows/tdd.md` | TDD 작업 시작, 중단, 완료 기준 |
| `workflows/review.md` | 리뷰와 자체 점검 기준 |

## 적용 우선순위

1. 프로젝트 루트의 `AGENTS.md`
2. 프로젝트 내부의 더 구체적인 영역별 지침
3. 이 `guardrails/` 디렉토리의 공통 기준
4. 개별 스킬의 `SKILL.md`

프로젝트 지침과 충돌하면 프로젝트 지침을 우선한다. 단, 검증 우회, 민감정보 노출, 사용자 변경 되돌리기 금지 같은 안전 규칙은 약화하지 않는다.

## 에이전트 사용법

- 작업 시작 전 대상 언어 문서를 하나 읽는다.
- 동작 변경, 버그 수정, 리팩터링은 `workflows/tdd.md`를 따른다.
- 설계나 리뷰 판단이 필요하면 `laws.md`에서 관련 법칙을 행동 규칙으로 적용한다.
- 작업 종료 전 `scripts/validate-plugin.ps1` 또는 `scripts/validate-plugin.sh`를 실행해 문서/어댑터 동기화가 깨지지 않았는지 확인한다.
