# 토론 프로토콜

## 전체 흐름

```
Phase 1: 독립 의견    →  Phase 2: 반박 라운드 (×N)  →  Phase 3: 종합
(병렬, 1회)              (병렬, 최대 3회)               (단일, 1회)
```

## Phase 1: 독립 의견 수집

### 실행 방법

Synthesizer를 제외한 모든 에이전트를 **동시에 병렬**로 스폰한다.

```
Agent 도구 호출 (병렬):
├── Agent 1: advocate 프롬프트
├── Agent 2: critic 프롬프트
├── Agent 3: ux-advocate 프롬프트
├── ...
└── Agent N-1: pragmatist 프롬프트
```

### 각 에이전트에게 전달하는 프롬프트

```markdown
## 토론 주제
{유저가 입력한 주제}

## 배경 정보
{유저가 제공한 추가 컨텍스트가 있다면}

## 너의 역할
{agents/*.md 에서 로드한 전체 페르소나 정의}

## 지시사항
- 너의 역할 관점에서 이 주제를 분석하라.
- 주장에는 반드시 구체적 근거를 제시하라.
- 결론을 명확히 내려라 (찬성/반대/조건부 찬성 등).
- 300단어 이내로 작성하라.

## 출력 형식
### {역할명}의 의견
**입장**: [한 줄 요약]
**근거**:
1. [근거 1]
2. [근거 2]
3. [근거 3]
**결론**: [명확한 결론]
```

## Phase 2: 반박 라운드

### 실행 조건

- `--rounds 1`: Phase 2 스킵, Phase 1 → Phase 3 직행
- `--rounds 2`: 반박 1회 → 종합
- `--rounds 3`: 반박 1회 → 재반박 1회 → 종합

### 각 라운드 프롬프트

```markdown
## 토론 주제
{주제}

## 너의 역할
{페르소나 정의}

## 이전 라운드 전체 의견
---
{Agent 1의 의견/반박}
---
{Agent 2의 의견/반박}
---
{Agent 3의 의견/반박}
---
... (모든 에이전트의 이전 라운드 출력)

## 지시사항
- 다른 에이전트들의 의견을 읽고, 너의 관점에서 반박하거나 보강하라.
- 동의하는 부분은 인정하되, 빠진 관점을 지적하라.
- 새로운 근거를 추가하라.
- 입장이 바뀌었다면 솔직히 인정하라.
- 200단어 이내로 작성하라.

## 출력 형식
### {역할명}의 반박 (라운드 {N})
**핵심 반박**: [가장 중요한 반박 포인트]
**동의 사항**: [동의하는 부분이 있다면]
**추가 근거**: [새로운 근거]
**수정된 입장**: [입장 변화가 있다면 명시, 없으면 "유지"]
```

### 라운드별 누적

- 라운드 2에서는 Phase 1 의견 + 라운드 1 반박을 모두 포함
- 라운드 3에서는 Phase 1 + 라운드 1 + 라운드 2를 모두 포함

## Phase 3: 종합

### 실행 방법

Synthesizer 에이전트를 **단일**로 스폰한다.

### 프롬프트

```markdown
## 토론 주제
{주제}

## 전체 토론 기록

### Phase 1: 독립 의견
{모든 에이전트의 초기 의견}

### Phase 2: 반박 라운드
#### 라운드 1
{모든 에이전트의 반박}
#### 라운드 2 (있다면)
{모든 에이전트의 재반박}

## 지시사항
- 모든 의견과 반박을 종합하여 분석하라.
- 갈등 지도를 작성하라 (누가 어떤 포인트에서 충돌하는지).
- 합의점을 추출하라.
- 2~3개 옵션을 구성하고, 각 옵션의 적합 조건을 명시하라.
- 최종 추천안을 선정하고 근거를 제시하라.
- references/report-template.md 형식을 따르라.
```
