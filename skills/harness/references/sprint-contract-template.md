# Sprint Contract Template

## 형식

```markdown
## Sprint {N} Contract: {스프린트 이름}

### 목표
{이번 스프린트에서 만들 것 — 1-2문장}

### Acceptance Criteria
1. [ ] {구체적, 테스트 가능한 기준}
2. [ ] {구체적, 테스트 가능한 기준}
3. [ ] {구체적, 테스트 가능한 기준}

### 이전 스프린트 피드백 (있을 경우)
- {이밸루에이터가 지적한 이슈와 수정 요청}

### 검증 방법
- 웹 앱: Playwright로 {구체적 테스트 시나리오}
- CLI: Bash로 {구체적 명령어와 기대 출력}

### 제약 사항
- AI 슬롭 금지 (slop-detection-guide.md 참조)
- 이전 스프린트 코드 깨뜨리지 않기
```

## 좋은 Acceptance Criteria

**좋은 예:**
- "시작 버튼 클릭 → 타이머가 25:00부터 카운트다운"
- "0:00 도달 → 알림음 + 휴식 모드 전환"
- "설정에서 시간 변경 → 다음 세션에 반영"

**나쁜 예:**
- "타이머가 잘 작동한다" (모호)
- "예쁜 디자인" (측정 불가)
- "React로 구현" (기술 디테일 — 플래너가 정하면 안 됨)
