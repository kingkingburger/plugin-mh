# PRD 출력 템플릿

ralph-prep 스킬이 생성하는 PRD 문서의 구조.

```markdown
# {주제} PRD

**작성일**: {YYYY-MM-DD}
**상태**: 완료
**작성 방법**: ralph-prep 스킬을 통한 심층 인터뷰

---

## 1. 개요

### 배경
{왜 이것을 만드는가}

### 목표
{무엇을 달성하려 하는가}

### 대상 사용자
{누가 사용하는가}

---

## 2. 유저 시나리오

### 시나리오 1: {시나리오명}

**전제 조건**: {사용자 상태, 환경}

**단계**:
1. {사용자가 ~를 한다}
2. {시스템이 ~를 보여준다}
3. {사용자가 ~를 선택한다}
4. ...

**예외 처리**:
| 예외 상황 | 대응 |
|-----------|------|
| {예외 1} | {어떻게 처리} |
| {예외 2} | {어떻게 처리} |

**화면 흐름** (선택):
```
┌─────────────┐         ┌─────────────┐
│  화면 A      │ ──────→ │  화면 B      │
│             │         │             │
│  [요소들]    │         │  [요소들]    │
└─────────────┘         └─────────────┘
```

### 시나리오 2: {시나리오명}
...

---

## 3. 의사결정 기록

| # | 결정 사항 | 선택지들 | 최종 선택 | 근거 | Phase |
|---|-----------|---------|-----------|------|-------|
| 1 | {무엇을 결정} | {A / B / C} | {선택} | {왜} | 1 |
| 2 | ... | ... | ... | ... | 2 |

---

## 4. 검증 계획

### 검증 환경
- **도구**: {Playwright / CLI / curl 등}
- **환경**: {로컬 / 스테이징 등}

### 정상 시나리오
| # | 시나리오 | 수행 단계 | 기대 결과 |
|---|---------|-----------|-----------|
| 1 | {시나리오} | {단계} | {기대 결과} |

### 엣지 케이스
| # | 시나리오 | 수행 단계 | 기대 결과 |
|---|---------|-----------|-----------|
| 1 | {이상한 패턴} | {단계} | {기대 결과} |

### 회귀 테스트
| # | 기존 기능 | 확인 사항 |
|---|-----------|-----------|
| 1 | {기능} | {깨지지 않는지 확인} |

  - [ ] {태스크 1}

### Phase 2: {이름}

## 6. Definition of Done

- [ ] 모든 정상 시나리오 통과
- [ ] 모든 엣지 케이스 통과
- [ ] 회귀 테스트 통과

- [ ] {프로젝트 특화 조건}

## 7. 참고 사항

- {추가 메모, 링크, 관련 문서 등}
```
