# /life-plan — 시즌별 5계층 인생 계획 코칭

mycraft 방법론을 다른 어떤 세션에서도 호출 가능하게 만든 코칭 스킬. 5계층(1년 방향 → 3개월 챕터 → 이번 달 시즌 → 이번 주 작전 → 오늘의 미션) + 발전 루프를 인터뷰 + 자질 검사 + 70-20-10 분배로 설계한다.

> Codex CLI 슬래시 커맨드로 호출되거나 자연어로 트리거된다. `$ARGUMENTS`에 사용자 입력이 주입된다.

## 핵심 원칙

> **시스템이 사용자를 끌고 가지 않는다. 좋은 질문을 던질 뿐이다.**

## 5계층 한 줄

| 한국어 | 영어 (내부) | 기간 |
|--------|------------|------|
| 1년 방향 | North Star | 6~12mo |
| 3개월 챕터 | Quarter Arc | 3mo |
| 이번 달 시즌 | Season | 1mo |
| 이번 주 작전 | Episode | 1wk |
| 오늘의 미션 | Mission | 1d |

## 워크플로우

### Step 0 — 호출 범위 + 저장 위치 확인

사용자에게 한 번에 묻는다:

1. 어느 레이어부터? (1년 방향 / 3개월 챕터 / 시즌 / 주 / 오늘)
2. 산출물 저장 위치는? (기본값 `secondBrain/12_ai_zone/mycraft/` / `_workspace/` / 사용자 지정 / 저장 안 함)

### Step 1 — 1년 방향 인터뷰 6문항 (해당 시)

한 번에 한 문항만 노출:

1. **시간 지평:** 지금 가장 견디기 어려운 것 한 줄.
2. **회피 프레이밍:** 그 상태가 1년 뒤에도 그대로면 어떤 모습?
3. **도달 프레이밍:** 1년 뒤 가장 자랑스러울 한 가지 변화는?
4. **통제권 점검:** 내 행동만으로 정의되는 버전은?
5. **첫 신호:** 1주 안 첫 신호는?
6. **안티골:** 절대 무너지면 안 되는 영역 1가지는?

목표 15분 이내. Q4·Q6는 필수, 나머지는 건너뛰기 허용.

### Step 2 — 5자질 검사

| 자질 | 약함 시 한 줄 메시지 |
|------|---------------------|
| 눈에 보이는가 | "달성을 누가 봐도 알 수 있는 신호 1가지는?" |
| 내 손에 있는가 | "내 행동만으로 정의되는 버전은?" |
| 한 문장이 되는가 | "사실 목표가 여러 개로 보입니다. 가장 중요한 1개부터." |
| 실패가 말이 되는가 | "언제까지, 무엇이 안 되면 실패?" |
| 내가 원해서 나왔는가 | "외부 압력 신호일 수 있습니다. 본인 언어로 다시?" |

점수 매기지 않음. 한 줄 메시지만.

### Step 3 — 3개월 챕터 분해

A. 거리 추정 (분기 1 / 2 / 4 / 자신없음)

B. 단계 결정 트리:
- 무너짐 → **회복**
- 새 루틴 있음 → **정착**
- 집중 부하 여유 → **돌파**
- 그 외 → **안정화**

C. 70-20-10 pillar 분배 (고정).

D. 첫 분기 보수 선택 권장.

### Step 4 — 시즌 win_condition 추상→측정 5단

1. 누가 봐도 → 2. 단위 → 3. 횟수·강도 → 4. 검증 가능 행동 → 5. 종료 조건.

**Almost Daily 원칙:** 매일 100% 강제 금지. "주 5일 / 4주 중 3주" 같은 buffer 내장.

### Step 5 — 주 작전

핵심 1 + 보조 1 + 버릴 행동 1 + 위험 신호 1 + 금요일 리뷰 3문항.

### Step 6 — 오늘 미션 2~3개

What + When + Where가 한 줄에. 1탭 완료 기본.

### Step 7 — 회고 4문항

| 문항 | 입력 |
|------|------|
| 사실 | 데이터 자동 채움 + 1줄 코멘트 |
| 놀람 | 자유 1~3줄 |
| 패턴 | 시스템 후보 2개 → 맞다/모르겠다/다른 |
| 다음 | 단 한 가지만 — 2줄 이상 차단 |

레이어별 변주:
- 주 4문항 5분 / 시즌 5문항 10분 / 분기 8문항 30분 / 연 5문항 + 인터뷰 재실시.

## 흔한 함정 8개

| 신호 | 메시지 |
|------|--------|
| 미션 평균 4개+ 2주 연속 | "1개 줄여볼까요?" |
| "해야 한다" | "외부 압력 신호." |
| "X보다" 비교 | "절대 상태로?" |
| "안 그러면" 공포 | "도달형도 적어두면 균형." |
| 눈에 보임·실패 둘 다 약함 | "첫 신호 1가지만 정해볼까요?" |
| win_condition 없이 시즌 시작 | (차단) "1줄 필수." |
| 같은 영역 3연속 미달 | "목표 크기·시간대 점검?" |
| 회고 3회 무시 | "1분 회고 모드?" |

같은 신호 시즌당 1번만.

## 산출물 저장

### 기본 경로

```
secondBrain/12_ai_zone/mycraft/
├── north-star/   YYYY-MM-DD.md
├── arcs/         YYYY-챕터N.md
├── seasons/      YYYY-MM.md
└── weekly/       YYYY-WNN.md
```

### Frontmatter 표준

```yaml
---
mycraft_id: <type>-<date>
created: YYYY-MM-DD
type: 1년-방향 | 3개월-챕터 | 이번-달-시즌 | 이번-주-작전
parent: <upper id> | null
status: 초안 | 진행중 | 완료
---
```

본문은 Obsidian `[[ ]]` 백링크로 상하위 연결.

### 권한

`secondBrain/12_ai_zone/` 외부는 read·write 모두 금지. 사용자 명시 지목 시에만 예외.

## 어휘 — 한국어 우선

사용자 노출 표현은 한국어. 영어는 괄호. WOOP·OKR 등 외부 프레임 용어도 한국어로 풀어서.

## 다른 커맨드와의 관계

| 상황 | 우선 |
|------|------|
| 요구 자체가 모호 | /vague |
| 사각지대 점검 | /unknown |
| 목표 너무 작음 | /moonshot |
| 3단계 깊이 문서 | /ouroboros |
| 다관점 결정 | /agent-arena 또는 /expert-review |

## 운영 규칙 (압축)

- 묻지 않는다 — 일간 안 묻기, 자동 채움 우선, 1초 침묵.
- 한 번에 한 가지 — 회고 "단 한 가지만" 강제.
- Almost Daily — 100% 강제 거부.
- 4주 미만 데이터로 추세 추론 금지.

## 출력 형식

종료 시 한 메시지로:

1. 저장된 파일 전체 경로 목록
2. Obsidian Graph 트리 (텍스트 다이어그램)
3. 자질 약함·모순 (있으면)
4. 다음 행동 1개

## Examples

```
User: "/life-plan 처음부터"
→ Step 0 (저장 위치 묻기)
→ Step 1 인터뷰 6문항
→ Step 2 자질 검사
→ Step 3 분기 챕터
→ Step 4~6 시즌 / 주 / 오늘
→ secondBrain/12_ai_zone/mycraft/{4파일} 저장
→ 트리 보고
```

```
User: "이번 분기 다시 짜줘 — 1년 방향은 그대로"
→ Step 1 건너뜀
→ Step 3부터 시작
→ arcs/2026-챕터N.md 갱신
```

```
User: "회고 코칭 — 이번 주만"
→ Step 7 4문항
→ weekly/2026-WNN.md에 회고 섹션 추가
```
