# life-plan: 시즌별 5계층 인생 계획 코칭

mycraft 프로젝트에서 도출한 다층 계획 방법론을 다른 어떤 세션에서도 호출 가능하게 만든 스킬. 5계층(1년 방향 → 3개월 챕터 → 이번 달 시즌 → 이번 주 작전 → 오늘의 미션) + 발전 루프(주/월/분기/연 회고)를 인터뷰 + 자질 검사 + 70-20-10 분배로 설계한다.

## 핵심 원칙

> **시스템이 사용자를 끌고 가지 않는다. 시스템은 좋은 질문을 던질 뿐이다.**

추천을 보여주되 답은 사용자가 한다. 약함 신호는 한 줄로만 — 잔소리 금지. 회고 "단 한 가지만 바꾼다" 강제로 변경 통제.

## When to Use

| 표현 | 언제 |
|------|------|
| "계획 짜자", "1년 방향", "북스타" | 처음부터 5계층을 새로 설계 |
| "이번 분기 재설계" | 3개월 챕터만 다시 (1년 방향은 유지) |
| "이번 시즌", "이번 달 시즌" | 1개월 시즌만 |
| "이번 주 작전" | 주간 에피소드만 |
| "회고 코칭" | 4문항 회고 진행 |
| "오늘 미션" | 오늘 1일분 미션 추출 |

상위 레이어 없이 하위만 호출하면 시스템은 상위 컨텍스트를 1줄로 요청한다 (없으면 가설로 채우고 사용자 확인).

## 5계층 한 줄 요약

| 한국어 | 영어 (내부) | 기간 | 핵심 필드 |
|--------|------------|------|----------|
| 1년 방향 | North Star | 6~12mo | title, anchor_date, pillars(1~3), north_metric, anti_goal |
| 3개월 챕터 | Quarter Arc | 3mo | theme(회복/정착/돌파/안정화), primary_pillar, season_slots(3) |
| 이번 달 시즌 | Season | 1mo | win_condition, parent_arc_id, arc_slot_index |
| 이번 주 작전 | Episode | 1wk | 핵심 챌린지 1 + 보조 1 + 버릴 행동 1 + 위험 신호 1 |
| 오늘의 미션 | Mission | 1d | 2~3개. 1탭 완료 가능. |

## 워크플로우

### Step 0 — 호출 범위 + 저장 위치 확인

`AskUserQuestion`으로 한 번에 묻는다:

1. 어느 레이어부터? (1년 방향 / 3개월 챕터 / 시즌 / 주 / 오늘)
2. 산출물 저장 위치는?
   - 기본값: `secondBrain/12_ai_zone/mycraft/`
   - 대안: `_workspace/` 또는 사용자 지정 경로
   - 저장 안 함 (대화로만)

### Step 1 — 1년 방향 인터뷰 (해당 시)

**6문항 순서** (한 번에 한 문항만 노출):

1. **시간 지평 정렬:** "지금 가장 견디기 어려운 것 한 줄. 1년 뒤에도 그대로면 진짜 곤란할 것."
2. **회피 프레이밍:** "그 상태가 1년 뒤에도 그대로면 어떤 모습일까요? 한 장면."
3. **도달 프레이밍:** "1년 뒤 가장 자랑스러울 한 가지 변화는?"
4. **통제권 점검:** "그 변화에 다른 사람·운이 끼어 있다면 빼고, 내 행동만으로 정의되는 버전은?"
5. **첫 신호:** "그 변화가 시작됐다고 말할 수 있는 1주 안 첫 신호는?"
6. **안티골:** "이 변화를 위해 절대 무너지면 안 되는 영역 1가지는?"

소요: 15분 이내가 목표. 사용자가 막히면 보조 질문으로 한 번 더, 그 다음은 건너뛰기 허용 (단 Q4·Q6는 필수).

### Step 2 — 5자질 검사 (모든 레이어 공통)

작성된 목표 문장에 대해 다음 5가지를 자동 체크. 약함은 한 줄 메시지만, 점수 매기지 않음.

| 자질 | 체크 휴리스틱 | 약함 시 메시지 |
|------|--------------|---------------|
| 눈에 보이는가 | 추상명사가 동사 자리에 있음 ("성장한다", "건강하다") | "이 목표가 달성되면 누가 봐도 알 수 있는 한 가지 신호는?" |
| 내 손에 있는가 | 외부 결과·타인 의존 ("조회수 1만") | "내 행동만으로 정의되는 버전은?" |
| 한 문장이 되는가 | "그리고/또한"이 2번 이상 | "사실 목표가 N개로 보입니다. 가장 중요한 1개부터." |
| 실패가 말이 되는가 | 마감 또는 실패 조건 부재 | "언제까지, 무엇이 안 되면 실패인가요?" |
| 내가 원해서 나왔는가 | "해야 한다", "남들이 다", "안 그러면" 등 | "이 표현은 외부 압력 신호일 수 있습니다. 본인 언어로 다시?" |

5개 중 5개 통과는 드물다. 1~2개 약함은 정상. 사용자는 무시할 수 있고 회고 때 다듬어도 된다.

### Step 3 — 3개월 챕터 분해 (해당 시)

**4단 사고 절차:**

A. **거리 추정** — "1년 방향까지 분기 몇 개의 거리?" (1 / 2 / 4 / 자신없음)

B. **단계(theme) 결정 트리:**

```
지금 가장 약한 영역이 무너져 있는가?
├─ 예 → 회복 (restore)
└─ 아니오. 안정적이지만 정체 →
   새 루틴이 있는가?
   ├─ 예 → 정착 (build)
   └─ 아니오. 강화할 것 →
      집중 부하 받을 여유 있는가?
      ├─ 예 → 돌파 (breakthrough)
      └─ 아니오. 굳히기 → 안정화 (consolidate)
```

C. **70-20-10 pillar 분배** (고정 — 50/30/20 같은 옵션 금지):
- **70%** primary_pillar — 거의 모든 시즌이 여기 정렬
- **20%** secondary — 보조
- **10%** maintenance — 유지만, 새 시도 금지

D. **첫 분기 보수 선택 안내** — "첫 분기는 욕심 줄여도 됩니다. 시즌 슬롯 3개 중 1개만 미리 정해도 충분."

### Step 4 — 시즌(월) win_condition — 추상→측정 5단

원문이 추상이면 5단계로 안내한다. 사용자가 어느 단계에서 멈춰도 OK (단 약함 표시).

1. 누가 봐도 알 수 있는 형태 → 2. 단위 → 3. 횟수·강도 → 4. 검증 가능한 행동 → 5. 시즌 종료 조건

**Almost Daily 원칙 강제:** 매일 100% 강제 win_condition 권장 안 함. "주 5일 / 4주 중 3주 / 30일 중 22일" 같은 buffer 내장형으로.

### Step 5 — 주(Episode) 작전

핵심 1 + 보조 1 + 버릴 행동 1 + 위험 신호 1 + 금요일 리뷰 3문항.

### Step 6 — 오늘 미션 2~3개

각 미션은 **What + When + Where**가 한 줄에. 1탭 완료 기본. 수치 입력은 카드 안에서.

### Step 7 — 회고 코칭 (호출 시)

**4문항 (모든 레이어 공통):**

1. **사실** — 데이터 자동 채움. 사용자는 1줄 코멘트.
2. **놀람** — 무엇이 예상과 달랐나? 자유 1~3줄.
3. **패턴** — 시스템 후보 2개 제시 (요일·시간대·미션 유형 중 강한 신호). "맞다/모르겠다/다른 패턴" 중 선택.
4. **다음** — 단 한 가지만 바꾼다면? **2줄 이상 입력 시 차단**, 1개 강제.

레이어별 변주:
- 주: 4문항 그대로. 5분 이내.
- 시즌: 4 + "한 줄 요약" 1.
- 분기: 4 + 분기 특화 4 (거리/theme 적합성/유지vs전환/anti_goal 침범).
- 연: 4 + "1년 방향 여전히 맞는가?" + 인터뷰 재실시.

## 흔한 함정 8개 — 자동 신호 + 한 번만 알림

| 신호 | 시스템 메시지 |
|------|--------------|
| 1주 미션 평균 4개 이상 2주 연속 | "이번 주 미션이 많습니다. 1개 줄여볼까요?" |
| "해야 한다" 표현 (인터뷰 단계) | "외부 압력 신호. 본인 언어로 다시?" |
| 비교 목표 ("X보다", 사람 이름) | "절대 상태로 바꾸어볼까요?" |
| 공포 목표 ("안 그러면") | "도달형 표현도 적어두면 균형이 잡힙니다." |
| 자질 5개 중 눈에 보임·실패 둘 다 약함 | "첫 신호 1가지만 정해볼까요?" |
| win_condition 없는 시즌 시작 시도 | (저장 차단) "win_condition 1줄 필수." |
| 같은 영역 시즌 3연속 미달 | "목표가 큰지, 시간대가 안 맞는지 점검해볼까요?" |
| 회고 알림 3회 무시 | "1분 회고(4문항 중 1·4번만) 모드 시도?" |

같은 신호는 시즌당 1번. "끄기" 누르면 영구 비활성.

## 산출물 저장 위치

### 기본값: `secondBrain/12_ai_zone/mycraft/`

```
secondBrain/12_ai_zone/mycraft/
├── north-star/   YYYY-MM-DD.md       ← 1년 방향
├── arcs/         YYYY-챕터N.md         ← 3개월 챕터

```

### 대안

호출 시 사용자가 다른 경로를 원하면 그 경로로. 옵션:
- `_workspace/` (mycraft 프로젝트 안에 있을 때 임시 작업용)
- `docs/specs/` (영구화 결정 시)
- 사용자 지정 경로

### Frontmatter 표준

```yaml
---
mycraft_id: <type>-<date or id>
created: YYYY-MM-DD
type: 1년-방향 | 3개월-챕터 | 이번-달-시즌 | 이번-주-작전
parent: <upper layer mycraft_id> | null
status: 초안 | 진행중 | 완료
---
```

본문에는 Obsidian `[[ ]]` 백링크로 상하위 연결. 사용자가 Obsidian Graph View에서 5계층 트리를 시각적으로 볼 수 있게.

### 권한 정책 엄수

- `secondBrain/12_ai_zone/` **외부는 절대 안 건드린다** (read도 안 함, write도 안 함).
- 사용자 본인이 명시적으로 외부 경로를 지목한 경우만 예외 — 그래도 변경 전 한 번 더 확인.

## 어휘 정책 — 한국어 우선

사용자에게 노출되는 모든 표현은 한국어 우선, 영어는 괄호로만.

| 영어 (내부 변수) | 한국어 (사용자에게 노출) |
|------------------|------------------------|
| North Star | 1년 방향 |
| Quarter Arc | 3개월 챕터 |
| Season | 이번 달 시즌 |
| Episode | 이번 주 작전 |
| Mission | 오늘의 미션 |
| restore / build / breakthrough / consolidate | 회복 / 정착 / 돌파 / 안정화 |
| Observable / Controllable / Expressible / Falsifiable / Self-aligned | 눈에 보이는가 / 내 손에 있는가 / 한 문장이 되는가 / 실패가 말이 되는가 / 내가 원해서 나왔는가 |

WOOP·OKR·Implementation Intention 같은 외부 프레임 차용 시에도 사용자에게는 영어 용어 노출 금지. 한국어로 풀어서.

## 다른 스킬과의 관계

| 상황 | 우선 스킬 |
|------|-----------|
| 요구사항 자체가 모호 | `vague` 먼저 → 명확해지면 `life-plan` |
| 1년 방향에 사각지대 점검 필요 | `unknown` (4분면 분석) |
| 1년 방향 목표가 너무 작아 보임 | `moonshot` (목표 상향) |
| 3단계(요구→설계→검증) 깊이 문서가 필요 | `ouroboros` (life-plan은 코칭, ouroboros는 문서 심화) |
| 회고 후 큰 결정 필요 | `agent-arena` 또는 `expert-review`로 다관점 |
| 1년 방향이 본업/생업 방향이라 시간 / 리스크 / 전략 검토 필요 | `tech-decision` 또는 `unknown` |

## 운영 규칙 (압축)

- **묻지 않는다 원칙** — 일간 화면은 안 묻기. 자동 채움 우선. 미션 완료 후 1초 침묵.
- **한 번에 한 가지** — 회고 4번 문항 "단 한 가지만"은 변경 통제. 2줄 이상 차단.
- **인지 부하 한계** — 주 회고 4문항·5분, 시즌 5문항·10분, 분기 8문항·30분.
- **Almost Daily** — 100% 강제 win_condition 거부.
- **데이터 부족 시** — 4주 미만 데이터로 추세 추론 금지. fallback은 사용자 직접 결정.

## 출력 형식 (호출 결과)

호출 종료 시 다음을 한 메시지로 보고:

1. 저장된 파일 전체 경로 목록
2. Obsidian Graph 상의 트리 구조 (텍스트 다이어그램)
3. 검증에서 발견한 자질 약함 또는 모순 (있으면)
4. 다음 행동 1개 (오늘 / 이번 주 / 이번 시즌 마감일)
