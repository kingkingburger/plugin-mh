# /life-plan — 평생 가치부터 오늘 미션까지: 6계층 인생 계획 코칭

mycraft 방법론을 어떤 세션에서도 호출 가능하게 만든 코칭 스킬. 6계층(평생 가치 → 1년 방향 → 3개월 챕터 → 이번 달 시즌 → 이번 주 작전 → 오늘의 미션) + 발전 루프를 인터뷰 + 자질 검사 + 70-20-10 분배로 설계한다. 사용자 노트는 응축 — `방향.md` 메인 + 월별 폴더 + "M월 N주차" 사람-친화 파일명.

> Codex CLI 슬래시 커맨드로 호출되거나 자연어로 트리거된다. `$ARGUMENTS`에 사용자 입력이 주입된다.

## 핵심 원칙

> **시스템이 사용자를 끌고 가지 않는다. 좋은 질문을 던질 뿐이다.**

회피 표현(안티골 등)은 기본 빼고, 사용자가 명시 요청 시에만.

## 6계층 한 줄

| 한국어 | 영어 (내부) | 기간 |
|--------|------------|------|
| 평생 가치 | Life Values | 평생 |
| 1년 방향 | North Star | 6~12mo |
| 3개월 챕터 | Quarter Arc | 3mo |
| 이번 달 시즌 | Season | 1mo |
| 이번 주 작전 | Episode | 1wk |
| 오늘의 미션 | Mission | 1d |

## 워크플로우

### Step 0 — 호출 범위 + 저장 위치

1. 어느 레이어부터?
2. 산출물 저장 위치? (기본값 `secondBrain/12_ai_zone/mycraft/` — `방향.md` + `YYYY-MM/M월 N주차.md`)

### Step 1A — 평생 가치 인터뷰 (1문항)

> "죽기 전까지 잃어버리지 않을 것 1~3개를 단어로."

명사 1개씩. 자질 검사는 "내가 원해서 나왔는가" 1개만.

### Step 1B — 1년 방향 인터뷰 (5문항)

1. 시간 지평: 지금 가장 견디기 어려운 것
2. 회피 프레이밍: 1년 뒤에도 그대로면 어떤 모습?
3. 도달 프레이밍: 가장 자랑스러울 변화는?
4. 통제권 점검 (필수): 내 행동만으로 정의되는 버전?
5. 첫 신호 (필수): 1주 안 첫 신호?

12분 이내. Q4·Q5 필수.

### Step 2 — 5자질 검사 (레이어별)

| 레이어 | 적용 |
|--------|------|
| 평생 가치 | "내가 원해서 나왔는가" 1개만 |
| 1년 방향 | "내가 원해서 나왔는가" 강함 + 나머지 약함 OK |
| 3개월 챕터 | 5개 중 3개 이상 |
| 이번 달 시즌 | 5개 모두 |
| 이번 주 작전 | 5개 모두 |
| 오늘의 미션 | 5개 (미션 1만 강제) |

5자질: 눈에 보이는가 / 내 손에 있는가 / 한 문장이 되는가 / 실패가 말이 되는가 / 내가 원해서 나왔는가.

### Step 3 — 3개월 챕터

거리 추정 → 단계(회복/정착/돌파/안정화) → 70-20-10 → 첫 분기 보수.

`방향.md`의 ## 분기 메모 섹션에 흡수.

### Step 4 — 시즌 win_condition (5단)

추상→측정 5단. Almost Daily 원칙.

### Step 5 — 주 작전

핵심 1 + 보조 1 + 버릴 1 + 위험 신호 1 + 금요일 3문항.

### Step 6 — 오늘 미션 2~3개

What + When + Where 한 줄. 4개 이상이면 함정.

### Step 7 — 회고 4문항

| 문항 | 입력 |
|------|------|
| 사실 | 데이터 자동 채움 + 1줄 |
| 놀람 | 자유 1~3줄 |
| 패턴 | 시스템 후보 2개 → 맞다/모르겠다/다른 |
| 다음 | 단 한 가지 — 2줄 이상 차단 |

레이어별: 주 4·5분 / 시즌 5·10 / 분기 8·30 / 연 5+인터뷰 재실시.

### Step 8 — 전날 밤 5단 (사용자 직접 운용)

| 단계 | 시간 | 내용 |
|------|------|------|
| 1. 가치 점검 | 30s | 평생 가치 셋 중 깎인 것? |
| 2. 회고 4문항 | 3min | 사실/놀람/패턴/단 한 가지 |
| 3. 이번 주 핵심 다시 보기 | 1min | |
| 4. 내일 미션 골격 | 3min | 3개 이하, What+When+Where |
| 5. 미션 1 자질 5체크 | 1min | 3개 이상 약하면 재작성 |

총 10~15분.

## 흔한 함정 7개

| 신호 | 메시지 |
|------|--------|
| 미션 4개+ 2주 연속 | "1개 줄여볼까요?" |
| "해야 한다" | "외부 압력?" |
| 비교 목표 | "절대 상태로?" |
| 눈에 보임·실패 둘 다 약함 | "첫 신호 1개?" |
| win_condition 없는 시즌 | (차단) |
| 같은 영역 3연속 미달 | "크기·시간대 점검?" |
| 회고 3회 무시 | "1분 모드?" |

## 산출물 저장 — 응축 정책

### 기본 구조

```
secondBrain/12_ai_zone/mycraft/
├── 방향.md                  ← 메인. 평생 가치 + 1년 + 분기 메모 + 점검
└── YYYY-MM/                 ← 월별 폴더
    ├── M월 1주차.md
    ├── M월 2주차.md
    └── ...
```

매월 새 `YYYY-MM/` 폴더, 매주 일요일 밤 새 `M월 N주차.md`.

### `방향.md` 구조 (풍부)

```
# 방향

## 평생 — 잃어버리지 않을 것
- **{값1}** — {한 줄 풀이}
- **{값2}** — {한 줄 풀이}
- **{값3}** — {한 줄 풀이}

## 1년 — {anchor_date}까지
> **{한 줄}**

### 영역 (70 / 20 / 10)
{영역 표 3행}

### 추적할 한 가지
{north_metric}

## 분기 메모
이번 분기 ({시작}~{종료}): **{theme}** 단계
{시즌 슬롯 3개 표}
분기 중 가드레일: {한 줄}

## 점검 일정
- {6개월 후}: 영역 비중 점검
- {1년 후}: 인터뷰 5문항 재실시
```

### `M월 N주차.md` 구조

```
# {N월 K주차} ({시작}~{종료})

## 핵심 한 가지 / 가드레일 / 금요일 리뷰 3문항

## 오늘 미션 (수시 갱신)
### {월-일 요일}
1~3 미션

## 전날 밤 체크리스트 (10분)
1~5단 압축

## 일별 일지 (이 주 누적)
### {요일}
[전날 밤 + 오늘 회고]
```

## 운영 규칙 (압축)

- 묻지 않는다 — 일간 안 묻기, 자동 채움 우선.
- 한 번에 한 가지 — 회고 "단 한 가지" 강제.
- Almost Daily — 100% 강제 거부.
- 4주 미만 데이터로 추세 추론 금지.
- 응축 산출물 — 방향.md + 월별 폴더 + "M월 N주차" 파일명.
- 회피 표현 금지 — 안티골 등 명시 요청 시에만.

## 출력 형식

종료 시:
1. 저장 파일 경로
2. 변경 섹션 요약
3. 자질 약함
4. 다음 행동 1개

## Examples

```
User: "/life-plan 처음부터"
→ 1A 가치 → 1B 5문항 → 2~6 → 방향.md + 2026-05/5월 1주차.md

User: "이번 분기 다시 짜줘"
→ Step 3부터 → 방향.md 분기 메모 갱신

User: "회고 코칭 이번 주만"
→ Step 7 → M월 N주차.md 일별 일지에 추가

User: "전날 밤 체크리스트 안내"
→ Step 8 5단 출력
```
