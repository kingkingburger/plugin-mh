# life-plan: 평생 가치부터 오늘 미션까지 — 6계층 인생 계획 코칭

mycraft 프로젝트에서 도출한 다층 계획 방법론을 다른 어떤 세션에서도 호출 가능하게 만든 스킬. 6계층(평생 가치 → 1년 방향 → 3개월 챕터 → 이번 달 시즌 → 이번 주 작전 → 오늘의 미션) + 발전 루프(주/월/분기/연 회고)를 인터뷰 + 자질 검사 + 70-20-10 분배로 설계한다. **사용자에게 노출되는 산출물은 의도적으로 응축한다** — 노트는 직관적 이름·날짜별 구조로, 정의·규칙은 시스템 안에만.

## 핵심 원칙

> **시스템이 사용자를 끌고 가지 않는다. 시스템은 좋은 질문을 던질 뿐이다.**

추천을 보여주되 답은 사용자가 한다. 약함 신호는 한 줄로만 — 잔소리 금지. 회고 "단 한 가지만 바꾼다" 강제로 변경 통제.

## 응축 원칙 (산출물 차원)

- **방향은 메인에 1개**, **주간은 월별 폴더 안에 사람-친화 이름**으로.
- 분기 챕터·시즌 같은 메타 정보는 사용자가 매일 안 보는 것이라 `방향.md` 안에 한 단락("분기 메모")으로만 흡수.
- 자질 5종 / theme 4종 / 70-20-10 같은 정의는 이 SKILL.md(시스템)에만 두고 사용자 노트에 옮기지 않는다.
- 회피·거부 표현(안티골 등)은 **사용자가 명시 요청한 경우에만** 노트에 넣는다. 기본은 빼고 본다.

## When to Use

| 표현 | 언제 |
|------|------|
| "삶의 가치", "잃기 싫은 것", "평생" | 0번째 — 평생 가치만 |
| "계획 짜자", "1년 방향", "북스타", "인생 계획" | 처음부터 6계층 |
| "이번 분기 재설계" | 3개월 챕터만 |
| "이번 시즌", "이번 달" | 1개월 시즌만 |
| "이번 주 작전" | 주간 에피소드만 |
| "회고 코칭" | 4문항 회고 |
| "오늘 미션", "내일 계획", "전날 밤" | 1일분 또는 전날 밤 5단 체크리스트 |

## 6계층 한 줄 요약

| 한국어 | 영어 (내부) | 기간 | 핵심 필드 |
|--------|------------|------|----------|
| **평생 가치** | Life Values | 평생 | 죽기 전까지 잃기 싫은 것 1~3개 (명사 단어). 다른 레이어가 깎으면 즉시 멈춤 |
| 1년 방향 | North Star | 6~12mo | title, anchor_date, pillars(1~3 + 70-20-10), north_metric |
| 3개월 챕터 | Quarter Arc | 3mo | theme(회복/정착/돌파/안정화), primary_pillar, season_slots(3) |
| 이번 달 시즌 | Season | 1mo | win_condition, parent_arc, arc_slot |
| 이번 주 작전 | Episode | 1wk | 핵심 1 + 보조 1 + 버릴 행동 1 + 위험 신호 1 |
| 오늘의 미션 | Mission | 1d | 2~3개. What+When+Where 한 줄. 1탭 완료 |

## 워크플로우

### Step 0 — 호출 범위 + 저장 위치 확인

`AskUserQuestion`으로 한 번에 묻는다:

1. 어느 레이어부터?
2. 산출물 저장 위치?
   - 기본값: `secondBrain/12_ai_zone/mycraft/` (응축 — `방향.md` + `YYYY-MM/M월 N주차.md`)
   - 대안: `_workspace/` 또는 사용자 지정 경로
   - 저장 안 함 (대화로만)

### Step 1A — 평생 가치 인터뷰 (1문항)

> "죽기 전까지 잃어버리지 않을 것 1~3개를 단어로 적어주세요."

- 4개 이상이면 "가장 본질적인 1~3개로 좁혀볼까요?"
- **명사 단어 1개씩** 강제 (형용사·문장 X). 예: "유머", "품위", "호기심".
- 자질 검사는 "내가 원해서 나왔는가" 1개만.
- 결과는 `방향.md` 최상단 ## 평생 — 잃어버리지 않을 것 섹션에 한 단어씩 + 가치별 한 줄 풀이.

이 인터뷰는 인생당 보통 1번.

### Step 1B — 1년 방향 인터뷰 (5문항)

한 번에 한 문항만 노출:

1. **시간 지평 정렬:** "지금 가장 견디기 어려운 것 한 줄. 1년 뒤에도 그대로면 진짜 곤란할 것."
2. **회피 프레이밍:** "그 상태가 1년 뒤에도 그대로면 어떤 모습일까요? 한 장면."
3. **도달 프레이밍:** "1년 뒤 가장 자랑스러울 한 가지 변화는?"
4. **통제권 점검 (필수):** "그 변화에 다른 사람·운이 끼어 있다면 빼고, 내 행동만으로 정의되는 버전은?"
5. **첫 신호 (필수):** "그 변화가 시작됐다고 말할 수 있는 1주 안 첫 신호는?"

소요 12분 이내. Q4·Q5 필수, 나머지는 건너뛰기 허용.

> 이전 버전의 "안티골 1문항"은 v1.2부터 기본 제거. 사용자가 필요로 하면 명시 요청 시에만 추가.

### Step 2 — 5자질 검사 (레이어별 적용)

자질 검사는 **레이어에 따라 적용 강도가 다르다**.

| 레이어 | 적용 자질 |
|--------|----------|
| 평생 가치 | "내가 원해서 나왔는가" 1개만 |
| 1년 방향 | "내가 원해서 나왔는가" 강함 + 나머지 4개는 약함 OK |
| 3개월 챕터 | 5개 중 3개 이상 통과 |
| 이번 달 시즌 | **5개 모두** |
| 이번 주 작전 | 5개 모두 |
| 오늘의 미션 | 5개 (미션 1만 강제, 2·3은 생략 가능) |

자질 약함은 한 줄 메시지만, 점수 매기지 않음.

| 자질 | 휴리스틱 | 약함 시 메시지 |
|------|----------|---------------|
| 눈에 보이는가 | 추상명사가 동사 자리 ("성장한다") | "달성을 누가 봐도 알 수 있는 한 가지 신호는?" |
| 내 손에 있는가 | 외부 결과·타인 의존 | "내 행동만으로 정의되는 버전은?" |
| 한 문장이 되는가 | "그리고/또한"이 2번 이상 | "사실 목표가 N개로 보입니다. 가장 중요한 1개부터." |
| 실패가 말이 되는가 | 마감/실패 조건 부재 | "언제까지, 무엇이 안 되면 실패?" |
| 내가 원해서 나왔는가 | "해야 한다", "남들이 다", "안 그러면" | "외부 압력 신호일 수 있습니다. 본인 언어로 다시?" |

### Step 3 — 3개월 챕터 분해

A. **거리 추정** — "1년 방향까지 분기 몇 개 거리?" (1 / 2 / 4 / 자신없음)

B. **단계(theme) 결정 트리:**

```
무너졌나? → 회복 (restore)
새 루틴 도입? → 정착 (build)
한 영역 집중 부하 여유? → 돌파 (breakthrough)
잘 굴러가는 것 굳히기? → 안정화 (consolidate)
```

C. **70-20-10 pillar 분배** (고정).

D. **첫 분기 보수 안내** — "시즌 슬롯 3개 중 1개만 미리 정해도 OK."

`방향.md` 안의 ## 분기 메모 섹션에 한 단락(theme + 시즌 슬롯 3개 표)으로 흡수.

### Step 4 — 시즌 win_condition (추상→측정 5단)

원문이 추상이면 5단 안내. 사용자가 어느 단계에서 멈춰도 OK.

1. 누가 봐도 → 2. 단위 → 3. 횟수·강도 → 4. 검증 가능 행동 → 5. 종료 조건.

**Almost Daily 원칙:** 100% 강제 금지. "주 5일 / 4주 중 3주" buffer 내장.

### Step 5 — 주 작전

핵심 1 + 보조 1 + 버릴 행동 1 + 위험 신호 1 + 금요일 리뷰 3문항.

### Step 6 — 오늘 미션 2~3개

각 미션 What + When + Where 한 줄. 1탭 완료. 4개 이상이면 함정 신호.

### Step 7 — 회고 코칭

**4문항 (전 레이어 공통):**

1. **사실** — 데이터 자동 채움 + 1줄 코멘트
2. **놀람** — 자유 1~3줄
3. **패턴** — 시스템 후보 2개 → 맞다/모르겠다/다른
4. **다음** — 단 한 가지 — **2줄 이상 차단**

레이어별: 주 4문항·5분 / 시즌 5문항·10분 / 분기 8문항·30분 / 연 5문항 + 인터뷰 재실시.

### Step 8 — 전날 밤 5단 (사용자 직접 운용)

사용자가 매일 밤 직접 운용. 시스템은 한 번 안내하고 멈춘다.

| 단계 | 시간 | 무엇 |
|------|------|------|
| 1. 가치 점검 | 30s | 오늘 평생 가치 셋 중 어느 하나 깎였나? (1줄) |
| 2. 회고 4문항 | 3min | 사실 / 놀람 / 패턴 / 단 한 가지 |
| 3. 이번 주 핵심 다시 보기 | 1min | `M월 N주차.md`의 핵심·위험·금요일 3문항 |
| 4. 내일 미션 골격 | 3min | 3개 이하, What+When+Where 한 줄씩 |
| 5. 미션 1 자질 5체크 | 1min | 3개 이상 약하면 다시 짜기 |

총 10~15분. 30분 넘으면 망설이고 있다는 신호 — 미션은 70% 확신이면 OK.

> 이전 버전의 Step 6(안티골 점검)은 v1.2부터 제거.

## 흔한 함정 7개 — 자동 신호

| 신호 | 메시지 |
|------|--------|
| 미션 평균 4개+ 2주 연속 | "1개 줄여볼까요?" |
| "해야 한다" (인터뷰) | "외부 압력 신호." |
| 비교 목표 ("X보다") | "절대 상태로?" |
| 자질 5개 중 눈에 보임·실패 둘 다 약함 | "첫 신호 1가지만?" |
| win_condition 없는 시즌 시작 | (차단) "1줄 필수." |
| 같은 영역 시즌 3연속 미달 | "목표 크기·시간대 점검?" |
| 회고 알림 3회 무시 | "1분 회고 모드?" |

같은 신호 시즌당 1번만.

## 산출물 저장 — 응축 정책

### 기본 구조

```
secondBrain/12_ai_zone/mycraft/
├── 방향.md                  ← 메인. 평생 가치 + 1년 방향 + 분기 메모 + 점검
└── YYYY-MM/                 ← 월별 폴더 (날짜별 분류, Obsidian Graph 정렬)
    ├── M월 1주차.md
    ├── M월 2주차.md
    └── ...
```

매월 시작 시 새 `YYYY-MM/` 폴더 생성. 매주 일요일 밤 새 `M월 N주차.md` 생성.

### `방향.md` 표준 구조

풍부함이 핵심. 단어 나열 금지, 한 줄 풀이 동반.

```markdown
# 방향

## 평생 — 잃어버리지 않을 것
- **{값1}** — {한 줄 풀이}
- **{값2}** — {한 줄 풀이}
- **{값3}** — {한 줄 풀이}

## 1년 — {anchor_date}까지
> **{한 줄}**

### 영역 (70 / 20 / 10)
| 영역 | 비중 | 이번 1년에 무엇을 |
|------|------|------------------|
| ... | 70% | ... |
| ... | 20% | ... |
| ... | 10% | ... |

### 추적할 한 가지
{north_metric 한 문장}

## 분기 메모
이번 분기 ({시작} ~ {종료}): **{theme}** 단계.
| 월 | 시즌 역할 |
|----|----------|
| {N월} | {역할} |
| ... | ... |

분기 중 가드레일: {한 줄}

## 점검 일정
- **{6개월 후}**: ... (가치는 그대로, 영역 비중만 점검)
- **{1년 후}**: 인터뷰 5문항 재실시 가능
```

### `M월 N주차.md` 표준 구조

```markdown
# {N월 K주차} ({시작} ~ {종료})

## 핵심 한 가지
{한 줄}

## 가드레일
{한 줄}

## 금요일 리뷰 3문항
1. ... 2. ... 3. ...

---

## 오늘 미션 (수시 갱신)
### {월-일 요일}
1. {미션 — What+When+Where, 분량}
2. ...
3. ...

---

## 전날 밤 체크리스트 (10분)
1. 가치 점검 — ...
2. 회고 4문항 — ...
3. 이번 주 핵심 다시 보기
4. 내일 미션 3개 — What+When+Where
5. 핵심 미션 자질 5체크

---

## 일별 일지 (이 주 누적)
### {월-일 요일}
[전날 밤 + 오늘 회고]
```

### 매주 일요일 밤 운용

- `M월 N주차.md`는 그대로 두고 새 `M월 (N+1)주차.md` 만든다 (지난 주 회고는 그 자리 그대로 보존).
- 새 달 시작 시 `YYYY-MM/` 폴더 새로 생성.

### 대안 경로

호출 시 사용자가 다른 경로 원하면 그 경로로:
- `_workspace/` (mycraft 프로젝트 안)
- `docs/specs/` (영구화)
- 사용자 지정

### 권한 정책 엄수

- `secondBrain/12_ai_zone/` 외부 read·write 금지.
- 사용자 명시 지목 시에만 예외.

## 어휘 — 한국어 우선

사용자 노출 표현은 한국어. 영어는 괄호 1회만.

| 영어 (내부) | 한국어 (노출) |
|-------------|--------------|
| Life Values | 평생 가치 |
| North Star | 1년 방향 |
| Quarter Arc | 3개월 챕터 |
| Season | 이번 달 시즌 |
| Episode | 이번 주 작전 |
| Mission | 오늘의 미션 |
| restore / build / breakthrough / consolidate | 회복 / 정착 / 돌파 / 안정화 |
| Observable / Controllable / Expressible / Falsifiable / Self-aligned | 눈에 보이는가 / 내 손에 있는가 / 한 문장이 되는가 / 실패가 말이 되는가 / 내가 원해서 나왔는가 |

WOOP·OKR·Implementation Intention 등 외부 프레임 사용자 노출 금지.

## 다른 스킬과의 관계

| 상황 | 우선 스킬 |

## 운영 규칙 (압축)

- **묻지 않는다 원칙** — 일간 안 묻기. 자동 채움 우선. 미션 완료 후 1초 침묵.
- **한 번에 한 가지** — 회고 4번 "단 한 가지만" 강제. 2줄 이상 차단.
- **인지 부하 한계** — 주 4문항·5분, 시즌 5·10, 분기 8·30.
- **Almost Daily** — 100% 강제 거부.
- **데이터 부족 시** — 4주 미만 데이터로 추세 추론 금지.
- **응축 산출물** — 방향.md 메인 + 월별 폴더 + 사람-친화 파일명.
- **회피 표현 금지** — 안티골 등 회피·거부 명시는 기본 빼고, 명시 요청 시에만.

## 출력 형식 (호출 결과)

종료 시 한 메시지로:

1. 저장된 파일 전체 경로
2. 변경된 섹션 요약
3. 자질 약함 (있으면)
4. 다음 행동 1개

## Examples

```
User: "/life-plan 처음부터"
→ Step 0 → 1A 평생 가치 → 1B 1년 방향 인터뷰 5문항
→ Step 2 자질(레이어별) → Step 3 분기 → Step 4~6 시즌·주·오늘
→ 방향.md + 2026-05/5월 1주차.md 작성

User: "이번 분기 다시 짜줘 — 1년 방향은 그대로"
→ Step 1 건너뜀 → Step 3부터 → 방향.md "분기 메모" 단락 갱신

User: "회고 코칭 이번 주만"
→ Step 7 4문항 → 2026-05/5월 N주차.md "일별 일지"에 추가

User: "전날 밤 체크리스트 안내"
→ Step 8 5단계 출력 → 사용자 직접 진행, 시스템은 안 묻기
```
