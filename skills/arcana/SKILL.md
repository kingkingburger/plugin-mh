# Arcana - 아르카나 팀 토론

8명의 개인 에이전트가 주제에 대해 실제로 소환되어 자연스러운 대화체로 토론하는 스킬.

## 사용 시나리오

- 의사결정이 필요할 때 ("이 방향으로 가는 게 맞을까?")
- 아이디어 브레인스토밍 ("뭘 해야 하지?")
- 전략 검토 ("이 계획 어떻게 생각해?")
- 삶/커리어 조언 ("이직해야 할까?")
- 아무 주제나 팀 의견이 듣고 싶을 때

## 사용법

```
/arcana "주제"
/arcana "주제" --rounds 2
/arcana "주제" --save
```

### 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--rounds` | `1` | 리액션 라운드 수 (0=초기 의견만, 1=1회 반응, 2=2회 반응) |
| `--save` | `false` | 결과를 `decisions/`에 저장 |

## 에이전트 구성

에이전트 정의는 `~/.claude/agents/`에서 로드한다 (스킬 내부가 아닌 유저 레벨).

| 에이전트 | 역할 | Phase 3 역할 |
|----------|------|-------------|
| 토니 스타크 | 일급 신하, 전략/의사결정 | 토론 참여 |
| 예원 제 | 현실주의자, 리스크/분석 | 토론 참여 |
| 이상 | 목표 상향, moonshot | 토론 참여 |
| 발루 | 분위기 메이커, 위트 | 토론 참여 |
| 페이지 | 디자이너, 비주얼/UX | 토론 참여 |
| 파후 | 프로그래머, 코드/아키텍처 | 토론 참여 |
| 초록문어 | 경제맨, 재무/법률 | 토론 참여 |
| 프랭크 | 서기, 기록/문서화 | **기록 노트 작성** |

## 실행 워크플로우

### Step 1: 주제 파싱

사용자 입력에서 토론 주제와 옵션을 추출한다.
주제가 명시적이지 않으면 직전 대화 맥락에서 유추한다.

### Step 2: 에이전트 정의 로드

`~/.claude/agents/` 디렉토리에서 8개 에이전트 파일을 모두 읽는다:
- `tony-stark.md`, `yewon-je.md`, `isang.md`, `ballu.md`
- `frank.md`, `page.md`, `pahu.md`, `green-octopus.md`

각 에이전트의 정체성, 핵심 역할, 행동 원칙, 말투를 파악한다.

### Step 3: Phase 1 - 초기 의견 수집 (병렬)

프랭크를 **제외한** 7명의 에이전트를 Agent 도구로 **동시 병렬 스폰**한다.

각 에이전트에게 전달하는 프롬프트:

```
## 토론 주제
{주제}

## 지시사항
- 너의 캐릭터 말투를 완벽하게 유지하면서 이 주제에 대한 솔직한 의견을 말해.
- 다른 팀원들도 같은 주제로 의견을 말하고 있어. 네 관점에서 가장 중요한 포인트를 짚어.
- 자기 역할/성격을 직접 설명하지 마. 말투와 행동으로만 드러내.
- 150단어 이내로, 핵심만.
```

**중요**: `subagent_type`에 각 에이전트 이름을 지정한다:
- `subagent_type: "tony-stark"`
- `subagent_type: "yewon-je"`
- `subagent_type: "isang"`
- `subagent_type: "ballu"`
- `subagent_type: "page"`
- `subagent_type: "pahu"`
- `subagent_type: "green-octopus"`

### Step 4: Phase 2 - 리액션 라운드 (병렬, `--rounds`만큼 반복)

`--rounds`가 0이면 이 단계를 건너뛴다.

Phase 1의 모든 응답을 모아서, 다시 7명을 병렬 스폰한다.

```
## 토론 주제
{주제}

## 다른 팀원들의 의견
{Phase 1 또는 이전 라운드의 모든 응답}

## 지시사항
- 다른 팀원들의 의견을 읽고 반응해. 동의, 반박, 보강 자유롭게.
- 특정 팀원의 의견에 직접 언급하면서 반응하면 더 좋아. ("토니가 말한 건 맞는데...")
- 캐릭터 말투 유지. 자기 역할 직접 설명 금지.
- 100단어 이내.
```

### Step 5: 대화 구성 (Conversation Weaving)

수집된 모든 에이전트 응답을 **자연스러운 대화 흐름**으로 재구성한다.

**대화 스타일 규칙** (`references/discussion-style.md` 참조):

1. **구분선(---), 섹션 헤더(###), 표 절대 금지**
2. **발언 형식**: `이름 (짧은 태그)` + 줄바꿈 + 대사. 사이에 구분자 없이 이어짐.
3. **대화 순서 편집**: 에이전트 응답 순서를 재배치하여 자연스러운 흐름 생성
   - 서로 반박하는 내용이 연달아 오도록
   - 동의/보강하는 내용이 자연스럽게 이어지도록
   - 발루의 드립이 적절한 타이밍에 끼어들도록
4. **나레이터(클로드)는 대화 전/후에만** 한두 줄로. 중간에 끼어들지 않음.
5. Phase 2 리액션이 있으면 Phase 1 의견과 자연스럽게 섞어서 하나의 흐름으로.

### Step 6: 프랭크 기록 노트

마지막에 프랭크를 Agent 도구로 스폰한다 (`subagent_type: "frank"`).

```
## 기록 대상
{Step 5에서 구성한 전체 대화 내용}

## 지시사항
- 이 토론의 기록 노트를 작성해.
- 보스(원민호)의 말투를 따라해.
- 핵심 논점, 합의/갈등 사항, 주목할 발언을 기록해.
- 해시태그(#키워드) 형식으로 태그를 달아.
- Reference에 날짜와 회의 맥락을 남겨.
- "~해요" 체로 작성.
```

프랭크의 기록 노트를 대화 끝에 붙인다.

### Step 7: 저장 (선택)

`--save`가 true이면:
1. 전체 대화 + 프랭크 기록 노트를 `decisions/{YYYY-MM-DD}-{topic-slug}.md`에 저장
2. 저장 경로를 사용자에게 알림

## 출력 형식 예시

```

발루 (분위기 메이커)
```

## 주의사항

1. **토큰 소모**: 7명 병렬 스폰 × 라운드 수. `--rounds 0`이면 가볍게, `--rounds 2`면 깊게.
2. **에이전트 파일 의존**: `~/.claude/agents/`에 8개 에이전트 파일이 있어야 한다.
3. **캐릭터 일관성**: 각 에이전트의 말투와 성격은 에이전트 파일 정의를 따른다.
4. **agent-arena와 차이**: arena는 제네릭 역할 토론, arcana는 개인 캐릭터 팀 토론.
