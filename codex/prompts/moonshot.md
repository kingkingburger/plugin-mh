# /moonshot — Moonshot: 최고 수준의 상향 목표 제안

사용자의 현재 목표를 입력받아, 목표 성격에 맞는 프레임워크를 적용하여 가장 높은 수준의 상향 목표 1개를 근거·액션플랜·리스크와 함께 제안한다. 목표 상향, 10x, BHAG, stretch goal, think bigger 등에서 트리거된다.

> Codex CLI 슬래시 커맨드로 호출되거나 자연어로 트리거된다. `$ARGUMENTS`에 사용자 입력이 주입된다.

## 핵심 원칙

1. **하나의 상향 목표만 제안**: 여러 단계가 아닌, 프레임워크가 제시하는 최고 수준 1개만 출력
2. **근거 기반**: 왜 이 수준이 가능한지 논리적 근거를 반드시 제시
3. **프레임워크 자동 선택 + 사용자 확인**: 목표 성격을 분석하여 최적 프레임워크를 추천하되, 사용자에게 확인
4. **범용성**: 업무, 개인, 학습, 프로젝트 등 모든 유형의 목표에 적용 가능

## 사용 시나리오

- "이번 분기 매출 목표 1000만원인데 더 높게 잡고 싶어"
- "하루 1시간 운동이 목표인데 이게 충분한지 모르겠어"
- "올해 안에 사이드 프로젝트 1개 완성하려고 하는데"
- "팀 생산성을 20% 올리고 싶은데 더 야심차게 가능할까?"

## 워크플로우

### Phase 0: 목표 캡처

사용자의 목표를 원문 그대로 기록한다. 자연어로 입력된 경우 핵심 요소를 추출:

- **목표 대상**: 무엇을 달성하려는가
- **현재 수준**: 지금 어디에 있는가 (있다면)
- **목표치**: 어디까지 가려는가
- **기간**: 언제까지인가 (있다면)
- **맥락**: 왜 이 목표를 세웠는가

### Phase 1: 맥락 파악

목표에서 누락된 핵심 정보를 사용자에게 질문하여 보완한다. **최대 2개 질문, 질문당 3-4개 옵션 제시.**

파악해야 할 핵심:

| 요소 | 필요한 이유 |
|------|-------------|
| 현재 수준 | 상향 폭 계산의 기준점 |
| 제약 조건 | 실현 가능성 판단 |
| 목표의 성격 | 프레임워크 선택 기준 |
| 시간 범위 | 단기 vs 장기 프레임워크 결정 |

**이미 충분한 정보가 있으면 이 단계를 건너뛴다.**

### Phase 2: 프레임워크 선택 및 확인

목표 성격을 분석하여 최적 프레임워크를 자동 선택한 뒤, **사용자에게 번호+라벨로 확인**받는다.

#### 프레임워크 선택 기준

| 목표 특성 | 추천 프레임워크 | 이유 |
|-----------|----------------|------|
| 수치/성과 기반, 단중기 | **10x Thinking** | 현재 성과의 10배 관점에서 사고 |
| 장기 비전, 5년+ | **BHAG** | 10-25년 대담한 목표 설정 |
| 미래 상태가 명확 | **Backcasting** | 이상적 미래에서 역산 |
| 근본적 재정의 필요 | **First Principles** | 기존 가정을 해체하고 재구성 |
| 단계적 확장 가능 | **Moonshot Ladder** | 현실적 → 야심적 → 문샷 단계 중 최상단 |

상세 프레임워크 가이드: `references/frameworks.md`

#### 확인 질문 예시

사용자에게 다음 질문을 제시하고 번호 또는 라벨로 답하게 하세요:

**목표 분석 결과, [10x Thinking] 프레임워크가 가장 적합합니다. 이 방향으로 진행할까요?** (프레임워크)
1. **[추천 프레임워크]** — [선택 근거 1줄] (추천)
2. **[차선 프레임워크]** — [이 프레임워크가 적합한 경우]
3. **직접 선택** — 5개 프레임워크 중 원하는 것을 지정

### Phase 3: 상향 목표 도출

선택된 프레임워크를 적용하여 **가장 높은 수준의 상향 목표 1개**를 도출한다.

도출 프로세스:
1. 프레임워크의 핵심 질문을 현재 목표에 적용
2. 기존 가정과 제약을 재검토
3. 가능한 최고 수준의 목표를 산출
4. 실현 가능성과 근거를 검증

### Phase 4: 최종 출력

두괄식으로 결과를 제시한다. 상세 템플릿: `references/output-template.md`

```markdown
# Moonshot 목표 제안

## 상향 목표
**[프레임워크명] 기반**: [상향된 목표 1줄]

> 원래 목표: [원본 목표]
> 상향 배율: [X배 / +X%]

## 왜 이 목표가 가능한가 (근거)
1. **[근거 1 제목]**: [설명]
2. **[근거 2 제목]**: [설명]
3. **[근거 3 제목]**: [설명]

## 액션플랜

### 즉시 (1-2주)
- [ ] [액션 1]
- [ ] [액션 2]

### 단기 (1-3개월)
- [ ] [액션 3]
- [ ] [액션 4]

### 중장기 (3개월+)
- [ ] [액션 5]
- [ ] [액션 6]

## 리스크 및 대응

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|--------|------------|--------|-----------|
| [리스크 1] | 높음/중간/낮음 | 높음/중간/낮음 | [대응] |
| [리스크 2] | ... | ... | ... |

## 성공 지표
- [측정 가능한 지표 1]
- [측정 가능한 지표 2]
```

## 주의사항

1. **현실성 유지**: 불가능한 목표가 아닌, 도전적이지만 근거 있는 목표를 제안
2. **맥락 존중**: 사용자의 상황과 제약을 무시하지 않음
3. **강요하지 않음**: 제안일 뿐, 최종 결정은 사용자의 몫
4. **재시도 가능**: 제안이 맞지 않으면 다른 프레임워크로 재시도 가능
